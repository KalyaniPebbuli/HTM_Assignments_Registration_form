History

HTML (HyperText Markup Language) is the standard language used to create and structure content on the web. Here's a quick overview of its history:

  1. 1989–1991: The Beginning
    • Inventor: Tim Berners-Lee, a physicist at CERN.
    • Goal:To enable researchers to share documents through hypertext on a network.
    • In 1991 , Berners-Lee published the first version of HTML with just 18 tags.

  2. 1995: HTML 2.0
    • Standardized by the Internet Engineering Task Force (IETF).
    • Included basic structural tags like <table>, <form> and improved support for web documents.

  3. 1997: HTML 3.2 and 4.01
    • HTML 3.2 (by W3C) introduced features like scripting (JavaScript), applets, and tables.
    • HTML 4.01 followed later in the same year with improved support for style sheets (CSS), better separation of content and design, and more form elements.

  4. 1999–2008: Rise of XHTML
    • XHTML (a stricter, XML-based version of HTML) was introduced but became less popular due to its rigidity and browser compatibility issues.

  5. 2008–2014: HTML5
    • Developed by WHATWG and W3C collaboratively.
    • Introduced powerful new elements like <video>, <audio>, <canvas>, and semantic tags like <article>, <section>.
    • Focused on richer web applications and multimedia support without plugins.

  6. 2014–Present: Living Standard
    • HTML is now maintained as a "living standard" by WHATWG.
    • It receives continuous updates to adapt to modern web needs (e.g., web components, accessibility improvements, and mobile support).

What is HTML

HTML stands for HyperText Markup Language . It is the standard language used to create webpages and web applications.

  • HyperText: Text that links to other text or documents (like clicking a link).
  • Markup Language: Uses tags to "mark up" or structure the content.

What Does HTML Do?
HTML is used to:
  • Define the structure of web pages.
  • Organize content into headings, paragraphs, lists, links, images, tables, and more.
  • Create forms to collect user input.
  • Embed media like audio and video.

Basic Structure of HTML Tag
<!DOCTYPE html>
<html>
  <head>
    <title>My First Web Page</title>
  </head>
  <body>
    <h1>Welcome to HTML</h1>
    <p>This is a simple paragraph.</p>
  </body>
</html>

Basic Tags Of HTML
Tag Desciption
<!DOCTYPE html> Defines the document type and version of HTML.
<html> Root element of an HTML page.
<head> Contains meta-information about the document (like title, character set).
<title> Sets the title of the document (shown in the browser tab).
<body> Contains the content of the document (text, images, links).

Why Learn HTML?
  • It's the foundation of all web development.
  • Required to create any website or online content.
  • Works hand-in-hand with CSS (for styling) and JavaScript (for interactivity).

Links And Images

Links
In HTML, links are created using the <a> tag (also called an anchor tag). Links allow users to navigate from one page to another, download files, or open email addresses directly.

Syntax of Links
<a href="www.google.com">google.com<a>
Attributes
Attributes provide additional information about the link. The most common attributes are:
  • href: Specifies the URL of the page the link goes to.
  • target: Specifies where to open the linked document (e.g., _blank ,_self,_parent,_top for a new tab).
  • title: Provides additional information about the link (shown as a tooltip).

Images
In HTML, images are embedded using the <img> tag. Images can be in various formats like JPEG, PNG, GIF, etc.

Syntax of Images
<img src="img_address" alt="image">
Attributes
Attributes provide additional information about the link. The most common attributes are:
  • src: Source – the file path or URL of the image
  • alt: Alternative text shown if image fails to load (important for accessibility)
  • Height: Sets the image size in pixels or percentage
  • Weight: Sets the image size in pixels or percentage

TABLES

Tables
HTML tables allow you to organize data into rows and columns, just like a spreadsheet. They are especially useful for displaying structured information such as schedules, price lists, or comparison charts.

Syntax
<table>
             <tr>
             <th>Header</th>
             <th>Header</th>
             </tr>
             <tr>
             <td>Data</td>
             <td>Data</td>
             </tr>
</table>;

Table Attributes
Attribute Description
border Sets the width (thickness) of the table border. Example: border="1"
Cellpadding Controls space inside each cell (padding between content & border).
Cellspacing Controls space between cells (padding between borders).


Attributes for <th> or <td> tags
Attribute Description
colspan Spans multiple columns (e.g., colspan="2" for 2 columns).
rowspan Spans multiple rows (e.g., rowspan="2" for 2 rows).
align Aligns content (left, center, right).
valign Aligns content vertically (top, middle, bottom).
width Sets the width of the cell (in pixels or percentage).
height Sets the height of the cell (in pixels or percentage).
bgcolor Sets background color of the cell (deprecated in HTML5; use CSS instead).

Forms

Forms
HTML forms are used to collect user input. They can include text fields, checkboxes, radio buttons, dropdowns, and more.

Syntax
<form action="submit.php" method="post">
             <input type="text" name="username">
             <input type="submit" value="Submit">
</form>

Form Attributes
Attribute Description
action URL where the form data will be sent.
method HTTP method to use (GET or POST).
enctype Encoding type for form data (e.g., multipart/form-data for file uploads).
target Where to display the response (e.g., _blank, _self).
Form Elements
Common form elements include:
  • <input>: Used for various input types (text, password, checkbox, radio, file). Syntax:
    <input type="text" name="username">
    <input type="password" name="password">

  • <label>: Associates a label with an input field, making it easier to identify and interact with
    Syntax:
    <label for="username">>Username</label>
    <input type="text" id="username">

  • <textarea>: Multi-line text input.
    Syntax:
    <textarea name="message"></textarea>

  • <select>: Dropdown list.
    Syntax:
    <select name="options">
                 <option value="1">Option 1</option>
    </select>

  • <button>: Button element.
    Syntax:
    <button type="submit">Submit</button>

  • <Radio button>:
    Syntax:
    <input type="radio name="gender" value="male">Male</input>

  • <Checkbox>: Checkbox input.
    Syntax:
    <input type="checkbox" name="subscribe" value="yes">Subscribe</input>

  • <File Upload>: File input.
    Syntax:
    <input type="file" name="file">

  • <Fieldset>:Fieldset groups related elements, and <legend> provides a caption for the fieldset.
    Syntax:
    <fieldset>
                 <legend>Personal Information</legend>
    </fieldset>

  • <Option Groups>: Used to group related options in a dropdown list.
    Syntax:
    <optgroup label="Group 1">
                 <option value="1">Option 1</option>
    </optgroup>

Semantic Tags
Semantic Tags
Semantic tags in HTML5 provide meaning to the content they enclose, making it easier for browsers and search engines to understand the structure and purpose of the content.

Examples of Semantic Tags
Some common semantic tags include:
  • <header>: Represents the header section of a document or section.
    Syntax:
    <header><h1>Welcome</h1></header>

  • <nav>: Defines a navigation section.
    Syntax:
    <nav><a href="#home">Home</a></nav>

  • <article>: Represents a self-contained piece of content.
    Syntax:
    <article><h2>Article Title</h2><p>Content</p></article>

  • <section>: Defines a thematic grouping of content.
    Syntax:
    <section><h2>Section Title</h2><p>Content</p></section>

  • <aside>: Represents content related to the main content (like a sidebar).
    Syntax:
    <aside><p>Related content</p></aside>

  • <footer>: Represents the footer section of a document or section.
    Syntax:
    <footer><p>Footer content</p></footer>


Benefits of Using Semantic Tags
Using semantic tags improves accessibility, SEO, and maintainability of web pages. They help search engines understand the content better and assist screen readers in providing context to users with disabilities.

Multimedia
Multimedia
HTML5 introduced native support for audio and video, allowing developers to embed multimedia content directly into web pages without relying on third-party plugins.

  • Audio Tag
    The <audio> tag is used to embed sound content in a web page.

    Syntax
    <audio controls>
                 <source src="audio.mp3" type="audio/mpeg">
    </audio>

  • Video Tag
    The <video> tag is used to embed video content in a web page.

    Syntax
    <video width="320" height="240" controls>
                 <source src="movie.mp4" type="video/mp4">
    </video>
  • Details Tag
    The <details> tag is used to create a disclosure widget from which the user can obtain additional information or controls.

    Syntax
    <details>
                 <summary>Click to see more</summary>
                 <p>Additional information here</p>
    </details>

Marquee Tag

Marquee Tag
The <marquee> tag is used to create scrolling text or images across a web page. However, it is deprecated in HTML5 and should be avoided in favor of CSS animations.

Syntax
<marquee behavior="scroll" direction="left">This is scrolling text</marquee>

Marquee_Tag_Attributes
Attribute Description
behavior Defines the scrolling behavior (scroll, slide, alternate).
direction Defines the direction of scrolling (left, right, up, down).
scrollamount Defines the speed of scrolling (in pixels).
scrolldelay Defines the delay between each scroll (in milliseconds).
loop Defines the number of times the marquee should loop (default is infinite).
width Defines the width of the marquee (in pixels or percentage).
height Defines the height of the marquee (in pixels or percentage).
